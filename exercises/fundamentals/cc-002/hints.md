# Hints for cc-002: Your First CLAUDE.md

## Hint 1

Think about what a new team member would need to know on their first day
working on this project. CLAUDE.md serves the same purpose — but for Claude.
Look at the project files to understand what you're documenting.

## Hint 2

Your CLAUDE.md needs at least: a project description section (use a `##` heading
with "Project" or "Description"), a commands section (use `##` with "Commands" or
"Development"), and it should mention the tech stack (TypeScript, Node.js, or npm).
Use markdown headings to structure it. Aim for 10+ lines.

## Hint 3

Create a CLAUDE.md with these sections:

```
## Project Description
A simple REST API for managing tasks — add, list, and complete tasks.

## Tech Stack
- TypeScript
- Node.js

## Commands
- `npm run build` — compile TypeScript
- `npm run start` — run the server
- `npm run dev` — watch mode
- `npm test` — run tests
```

Make sure it's at least 10 lines total.
